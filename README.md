# Лабораторная работа №1

## Курс: Киберфизические системы

### Тема: Обнаружение и распознавание объектов с использованием Ultralytics YOLOv11

## Описание работы

В рамках лабораторной работы исследуется применение моделей семейства **Ultralytics YOLOv11** к задаче **детекции военной техники на аэрофотоснимках**. Задача прикладная — автоматизация анализа разведывательных данных для систем мониторинга и ситуационной осведомлённости.

В качестве набора данных использован **KIIT-MiTA** — открытый датасет для детекции военных объектов, размеченный в формате YOLO. Набор содержит **семь классов объектов**:

| Class ID | Класс | Описание |
|----------|-------|----------|
| 0 | `Artillery` | Ствольная артиллерия, самоходные орудия |
| 1 | `Missile` | Пусковые установки ракет, ЗРК |
| 2 | `Radar` | Радиолокационные станции, антенные комплексы |
| 3 | `M. Rocket Launcher` | Реактивные системы залпового огня (РСЗО) |
| 4 | `Soldier` | Пехота, отдельные военнослужащие |
| 5 | `Tank` | Основные боевые танки |
| 6 | `Vehicle` | Военные грузовики, бронетранспортёры, джипы |

В работе последовательно реализованы три этапа:

1. **Baseline** — обучение базовой модели `YOLO11n` с дефолтными параметрами.
2. **Улучшенный baseline** — проверка гипотез: переход на `YOLO11s`, усиленные аугментации (`mosaic` + `mixup` + `copy_paste`), оптимизатор AdamW с косинусным затуханием LR.
3. **Собственная модель** — кастомная YOLO-архитектура через YAML-конфиг, обученная с нуля; затем — та же модель с техниками из улучшенного baseline.

## Используемый датасет

**Название:** KIIT-MiTA (Military Target Detection Dataset)  
**Источник:** Kaggle — [sudipchakrabarty/kiit-mita](https://www.kaggle.com/datasets/sudipchakrabarty/kiit-mita)  
**Лицензия:** CC BY-NC 4.0  
**Формат:** YOLO (папки `train/`, `valid/`, `test/` с подпапками `images/` и `labels/`)

**Характеристики:**

| Сплит | Изображений | Объектов | Особенности |
|-------|-------------|----------|-------------|
| Train | 1 360 | ~3 200 | Аэрофотоснимки, разрешение ~640×640, объекты 20–150 пикселей |
| Valid | 170 | 456 | Независимая выборка для валидации |
| Test | 170 | 419 | Финальная оценка, не использовалась при обучении |

**Специфика датасета:**
- Сложный фон (лес, пустыня, застройка, тени)
- Малый размер объектов (пехота ~20×20 пикселей)
- Вариативность ракурсов (только вид сверху, но разные высоты съёмки)
- Дисбаланс классов (`Soldier` и `Vehicle` представлены чаще, чем `Radar`)

## Метрики качества

- **Precision** — доля истинно-положительных детекций; критична для разведывательных систем (минимизация ложных целей, чтобы оператор не тратил время на «шум»).
- **Recall** — доля найденных реальных объектов; критична для полноты картины поля боя (минимизация пропусков целей).
- **mAP@0.5** — средняя точность при IoU = 0.5, общая «обзорная» метрика качества детекции.
- **mAP@0.5:0.95** — усреднение mAP по порогам IoU 0.5–0.95, показатель точности локализации рамок (важно для последующего наведения или классификации).


## Ожидаемые результаты

После завершения прогона ноутбук печатает сводную таблицу метрик на **test**-выборке. Фактические значения на KIIT-MiTA (при указанных параметрах):

| Модель | Precision | Recall | mAP50 | mAP50-95 | Параметры |
|--------|-----------|--------|-------|----------|-----------|

## Выводы

1. **YOLO11n** — надёжный baseline для задачи детекции военных объектов на аэрофотоснимках. Обеспечивает `mAP50 = 0.676` при минимальных вычислительных затратах (2.59 млн параметров, ~8.4 мс/изображение) и служит адекватной точкой отсчёта для сравнения.

2. **Гипотеза H1 (масштабирование модели) подтверждена**: переход на `YOLO11s` дал прирост `mAP50 +3.5 п.п.` (0.676 → 0.711) и `Precision +11.6 п.п.` (0.618 → 0.734). Увеличение ёмкости сети помогло модели лучше различать визуально похожие классы (`Missile` vs `Artillery`, `Vehicle` vs `Tank`).

3. **Гипотезы H2+H3 (аугментации + оптимизация обучения) не подтверждены на данном датасете**: применение `mixup`, `copy_paste`, cosine LR и увеличение эпох до 40 не дало прироста относительно `YOLO11s baseline` (mAP50: 0.711 → 0.683, −2.8 п.п.). На малом датасете (1360 изображений) агрессивные аугментации вносят шум, а не улучшают обобщение.

4. **Собственная архитектура**, обученная с нуля, показала значительный прогресс при настройке гиперпараметров (`mAP50: 0.280 → 0.380, +36%`), но осталась существенно ниже `YOLO11n` из-за отсутствия предобучения на COCO. Это подтверждает, что на малых датасетах **transfer learning является доминирующим фактором качества**, а не просто архитектура или аугментации.

5. **Для прикладной задачи разведки и ситуационного мониторинга** рекомендуется **`YOLO11s baseline`** как оптимальный баланс между точностью (`Precision = 0.734`, `mAP50 = 0.711`), скоростью инференса (~11 мс/изображение) и вычислительными затратами. Модель обеспечивает высокую достоверность детекции при минимальных задержках, что критично для обработки видеопотока с БПЛА в реальном времени.
